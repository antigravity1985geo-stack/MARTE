## 1. Migration

```sql
\i create_medical_billing.sql
```

**8 ახალი ცხრილი:**
- `insurance_companies` — სადაზღვ. კომპანიები
- `patient_insurance` — პაციენტის პოლისი (coverage %, deductible, limits)
- `medical_invoices` — ინვოისი header + insurance split + computed `balance_due`
- `invoice_items` — line items (procedure, qty, price, discount, tax, is_insured)
- `invoice_payments` — გადახდების ჩანაწ. (trigger auto-updates invoice balance + status)
- `insurance_claims` — claim header (submitted/approved/rejected/paid)
- `installment_plans` — განვადების პარამეტრები
- `installment_schedule` — auto-generated payment schedule

**Auto triggers:**
- `INV-2026-00042` / `CLM-2026-00018` — sequential numbers
- `update_invoice_on_payment` — payment INSERT → invoice.amount_paid + status auto-update
- `mark_overdue_invoices()` — run as cron or on-demand

**Functions:**
- `create_installment_schedule(plan_id)` — generates schedule rows per frequency

## Invoice Flow

```
Draft → Issue → Partially Paid → Paid
                    ↓              ↓
                  Overdue        Done
```

## Insurance (co-pay) Split

```
Invoice Total: ₾500
Insurance (80%): ₾400   → Insurance Claim → paid by insurer
Co-pay (20%):   ₾100   → Patient pays directly
```

`is_insured` per line item — surgical items may be covered, consultation may not be.

## Installment Plan

```
Balance: ₾900
Down payment: ₾300 → recorded as cash payment immediately
Financed: ₾600 / 3 months = ₾200/month
Schedule: auto-generated by create_installment_schedule() RPC
```

## Insurance Claim Workflow

```
draft → submitted → under_review → approved → paid
                                 ↘ partial
                                 ↘ rejected → resubmitted → …
```

## UI Features

### Invoices tab
- KPI strip: collected / outstanding / overdue count / total
- Status filter + search
- List → click → side panel
- Detail panel: payment progress bar, insurance split, line items, payment history
- Actions: Issue / Record Payment / Installment / Create Claim / Print

### Claims tab  
- KPI: claimed / approved / paid totals
- Status filter chips
- Submit draft claim with one click
- Mark approved claims as paid

### Payment modal
- Balance + co-pay remaining shown
- Cash / Card / Bank Transfer
- Auto-updates invoice status via DB trigger

### Installment modal
- Down payment input
- Count presets: 2/3/6/9/12
- Weekly / biweekly / monthly
- Live preview: financed amount + per-installment
- Schedule auto-generated via RPC
